# Концепция приложения и система взаимодействия мастера с игроками

## Главная идея проекта

Это приложение **не должно заменять D&D**.

Игроки продолжают играть за столом, использовать настоящие кубики, миниатюры, карты и общаться друг с другом.

Приложение должно убрать всю рутину:

* ведение карточки персонажа;
* автоматический расчет характеристик;
* учет предметов;
* учет эффектов;
* учет состояний;
* учет заклинаний;
* учет опыта;
* учет инвентаря;
* расчет модификаторов;
* автоматическое изменение характеристик при любых изменениях.

То есть приложение — это **интерактивная карточка персонажа**, а не виртуальный стол.

---

# Роли

## Игрок

Игрок полностью управляет своим персонажем.

Он может:

* изменять текущее HP;
* получать лечение;
* получать временные HP;
* отмечать использованные ячейки заклинаний;
* делать короткий и длинный отдых;
* использовать способности;
* экипировать предметы;
* пользоваться инвентарем;
* смотреть активные эффекты;
* получать опыт;
* повышать уровень (если разрешено мастером);
* бросать цифровые кубики (если включена эта функция);
* либо использовать настоящие кубики.

---

## Мастер

Мастер **не редактирует характеристики игроков напрямую**.

Он управляет происходящими событиями.

Основная задача мастера — создавать события, которые автоматически воздействуют на персонажей.

Например:

* дракон использовал устрашающее присутствие;
* маг наложил Bless;
* шаман наложил Bane;
* персонажи попали в ядовитое облако;
* все получили урон от ловушки;
* все получили лечение;
* всем начислен опыт;
* игрок получил новый предмет;
* персонажи получили временные HP.

---

# Главное изменение

Мастер НЕ изменяет:

* Силу
* Ловкость
* Мудрость
* Класс брони
* Навыки
* Бонусы

Все изменения происходят только через систему событий и эффектов.

---

# Система событий

У мастера должна быть отдельная вкладка:

## Создать событие

Тип события:

* Урон
* Лечение
* Выдать опыт
* Выдать предмет
* Применить состояние
* Применить заклинание
* Добавить эффект
* Снять эффект
* Выдать вдохновение
* Начать отдых
* Закончить отдых

После выбора события мастер выбирает цели.

Например:

Все игроки

или

Конкретный персонаж.

---

# Автоматические проверки

Например:

Мастер выбирает:

Дракон → Frightful Presence

Приложение знает:

Спасбросок Мудрости

СЛ 18

Игрок получает уведомление:

"Сделайте спасбросок Мудрости.

Ваш бонус: +6"

Если используются реальные кубики:

Игрок бросает настоящий кубик.

Вводит результат.

Приложение автоматически считает итог.

Если используется цифровой кубик:

Приложение бросает автоматически.

После этого автоматически определяется:

Успех

или

Провал

Если провал —

эффект применяется автоматически.

Мастер получает итог по всем игрокам.

---

# Автоматический расчет

Приложение никогда не хранит вычисляемые значения.

Например:

Класс брони вычисляется из:

броня

щит

магические предметы

активные эффекты

заклинания

расы

класса

домашних правил

То же самое относится к:

* инициативе;
* спасброскам;
* навыкам;
* скорости;
* максимальному HP;
* модификаторам характеристик;
* бонусам атаки.

---

# Источник каждой характеристики

При нажатии на любое значение игрок должен видеть:

Например:

Сила = 18

15 Базовое значение

+2 Расовый бонус

+1 Магический пояс

=18

Или:

Класс брони = 21

18 Латы

+2 Щит

+1 Плащ защиты

=21

Игрок всегда понимает, откуда взялось итоговое значение.

---

# Журнал событий

Все изменения автоматически записываются.

Например:

Получен эффект Bless

Использована ячейка 2 уровня

Получено 14 урона

Получено лечение 8 HP

Получен предмет

Получен опыт

Эффект закончился

Это позволит всегда восстановить историю.

---

# Система состояний

Мастер должен иметь готовый список стандартных состояний D&D.

Минимальный список:

* Blinded (Ослеплён)
* Charmed (Очарован)
* Deafened (Оглох)
* Frightened (Испуган)
* Grappled (Схвачен)
* Incapacitated (Недееспособен)
* Invisible (Невидим)
* Paralyzed (Парализован)
* Petrified (Окаменел)
* Poisoned (Отравлен)
* Prone (Сбит с ног)
* Restrained (Опутан)
* Stunned (Ошеломлён)
* Unconscious (Без сознания)

# Настройки кампании

В кампании должна быть настройка:

Использовать настоящие кубики
или
Использовать цифровые кубики.

Если включены реальные кубики —
приложение никогда не бросает кубики самостоятельно.
Оно только автоматически считает итоговый результат с учетом всех бонусов и штрафов.

---

# Основная цель проекта

Приложение не должно играть вместо игроков.
Оно должно избавить игроков и мастера от всей рутинной математики, автоматизировать подсчет характеристик, отслеживать эффекты и помогать вести персонажа максимально удобно, при этом полностью сохраняя атмосферу классической настольной игры с реальными кубиками, миниатюрами и живым общением за столом.

Мастер может выбрать эффект из стандартных правил D&D, но также создать собственный.
Например:
Название: Проклятие древнего леса
Тип: Дебафф
Влияние: −2 к проверкам Ловкости, скорость уменьшена на 3 м
Длительность: До снятия проклятия
Концентрация: Нет

После сохранения этот эффект появляется в библиотеке кампании и его можно применять повторно одним нажатием. Это позволит использовать не только официальные состояния и заклинания, но и любые авторские механики, не меняя код приложения. Для кампаний с домашними правилами это будет очень полезно.
